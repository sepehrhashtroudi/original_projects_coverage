16    /**
17    * Specialized POJO serializer that differs from
18    * {@link com.fasterxml.jackson.databind.ser.BeanSerializer}
19    * in that instead of producing a JSON Object it will output
20    * a JSON Array, omitting field names, and serializing values in
21    * specified serialization order.
22    * This behavior is usually triggered by using annotation
23    * {@link com.fasterxml.jackson.annotation.JsonFormat} or its
24    * equivalents.
25    *<p>
26    * This serializer can be used for "simple" instances; and will NOT
27    * be used if one of following is true:
28    *<ul>
29    * <li>Unwrapping is used (no way to expand out array in JSON Object)
30    * </li>
31    * <li>Type information ("type id") is to be used: while this could work
32    * for some embedding methods, it would likely cause conflicts.
33    * </li>
34    * <li>Object Identity ("object id") is used: while references would work,
35    * the problem is inclusion of id itself.
36    * </li>
37    *</ul>
38    * Note that it is theoretically possible that last 2 issues could be addressed
39    * (by reserving room in array, for example); and if so, support improved.
40    *<p>
41    * In cases where array-based output is not feasible, this serializer
42    * can instead delegate to the original Object-based serializer; this
43    * is why a reference is retained to the original serializer.
44    *
45    * @since 2.1
46    */
