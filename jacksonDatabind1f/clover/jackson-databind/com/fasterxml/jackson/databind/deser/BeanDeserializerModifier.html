15    /**
16    * Abstract class that defines API for objects that can be registered
17    * to participate in constructing {@link JsonDeserializer} instances
18    * (via {@link DeserializerFactory}).
19    * This is typically done by modules that want alter some aspects of deserialization
20    * process; and is preferable to sub-classing of {@link BeanDeserializerFactory}.
21    *<p>
22    * Note that Jackson 2.2 adds more methods for customization; with earlier versions
23    * only {@link BeanDeserializer} instances could be modified, but with 2.2 all types
24    * of deserializers can be changed.
25    *<p>
26    * Sequence in which callback methods are called for {@link BeanDeserializer} is:
27    * <li>{@link #updateProperties} is called once all property definitions are
28    * collected, and initial filtering (by ignorable type and explicit ignoral-by-bean)
29    * has been performed.
30    * <li>{@link #updateBuilder} is called once all initial pieces for building deserializer
31    * have been collected
32    * </li>
33    * <li>{@link #modifyDeserializer} is called after deserializer has been built
34    * by {@link BeanDeserializerBuilder}
35    * but before it is returned to be used
36    * </li>
37    * </ol>
38    *<p>
39    * For other types of deserializers, methods called depend on type of values for
40    * which deserializer is being constructed; and only a single method is called
41    * since the process does not involve builders (unlike that of {@link BeanDeserializer}.
42    *<p>
43    * Default method implementations are "no-op"s, meaning that methods are implemented
44    * but have no effect; this is mostly so that new methods can be added in later
45    * versions.
46    */
