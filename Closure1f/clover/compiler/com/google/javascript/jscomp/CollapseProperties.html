44    /**
45    * Flattens global objects/namespaces by replacing each '.' with '$' in
46    * their names. This reduces the number of property lookups the browser has
47    * to do and allows the {@link RenameVars} pass to shorten namespaced names.
48    * For example, goog.events.handleEvent() -> goog$events$handleEvent() -> Za().
49    *
50    * <p>If a global object's name is assigned to more than once, or if a property
51    * is added to the global object in a complex expression, then none of its
52    * properties will be collapsed (for safety/correctness).
53    *
54    * <p>If, after a global object is declared, it is never referenced except when
55    * its properties are read or set, then the object will be removed after its
56    * properties have been collapsed.
57    *
58    * <p>Uninitialized variable stubs are created at a global object's declaration
59    * site for any of its properties that are added late in a local scope.
60    *
61    * <p>If, after an object is declared, it is referenced directly in a way that
62    * might create an alias for it, then none of its properties will be collapsed.
63    * This behavior is a safeguard to prevent the values associated with the
64    * flattened names from getting out of sync with the object's actual property
65    * values. For example, in the following case, an alias a$b, if created, could
66    * easily keep the value 0 even after a.b became 5:
67    * <code> a = {b: 0}; c = a; c.b = 5; </code>.
68    *
69    * <p>This pass doesn't flatten property accesses of the form: a[b].
70    *
71    * <p>For lots of examples, see the unit test.
72    *
73    */
