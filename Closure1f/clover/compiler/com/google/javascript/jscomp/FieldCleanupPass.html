57    /**
58    * Search for fields to cleanup by looking for nodes in the tree which are
59    * root nodes of qualified names and getting the final token of the qualified
60    * name as a candidate field.
61    * <p>
62    * Once a candidate field is found, ask the {@code JSTypeRegistry} for all
63    * JSTypes that have a field with the same name, and check if the field on
64    * that type is defined in the file the compiler is cleaning up. If so, remove
65    * the field, and update the {@code JSTypeRegistry} to no longer associate the
66    * type with the field.
67    * <p>
68    * This algorithm was chosen for simplicity and is less than optimally
69    * efficient in two ways:
70    * <p>
71    * 1) All types with a matching field name are iterated over (when only types
72    * that extend or implement the JSType indicated by the containing object in
73    * the found Qualified Name need to be checked).
74    * <p>
75    * 2) All Qualified Names are checked, even those which are not L-Values or
76    * single declarations of an Type Expression. In general field should only be
77    * declared as part of an assignment ('ns.Type.a = 3;') or stand alone name
78    * declaration ('ns.Type.a;').
